# 说明


* 风机多人在线音频标注管理平台


### 音频标注的困难

* 没有现成免费的工具

* 标注的时候最好听一下音频，要多个文件一起参考

* 数据集没地方去管理，标注也没有好的方案去及时查看。比我我现在想查看一下 A 标签是怎么标注的，要打开 labelimg 一个个翻找到 A 标签

* 当前的数据库只能管理 wav，不能管理音频的标注，

### 之前方案的问题

* 将标注的图像和标注声音事情搞乱了，mel 图是原始音频的信息的压缩，我们的最终的参考对象还是声音，所以在标注的时候一定要能听到之前的声音，声音是最终的参考，而不是图像

* 之前的方案标注的时候没法听到标注方框对应的各点位的声音

* 数据集存放混乱，每个人标注的结果中间互不相通

* 不适用图像标注的方案（ucd）的原因，ucd 一套方案中，默认图像是信息的最底层，但是在声纹的标注中，声音才是最底层，图像只是方便使用 detect 这套标注方案的中间结果，

* ucd 方案是基于数据库进行交互的，当前的设计方案是基于 api 进行交互的，数据库这一层被包装起来了

* 当前的 mel 图只是一种声音可视化的方式，后续我们可能有别的声音可视化的方式，当前的方案不好多个可视化结果互相切换辅助标注

* 之前的方案不好记录每一个框是谁标注的，

* 标签名，文件名，都没有强制标准化

### 当前的方案

* 6wav + position_id 自动生成一个唯一的 buc (blade uc)

* 6 * wav + fuc -> img， img 不单独给一个 uniquecode（uc），图片的编号由音频的编号 + 生成图片的方法组成 

* 支持多人在线编辑，当一个人在标注某一个 buc 时，其他人无法标注同一个图片

* 可以定制多个数据集，每个人标注指定数据集的方式可以多人协作

* 提供 api + 鉴权的 方式在线开放 xml img wav 文件的下载


### 使用流程

* 数据入库，入库数据需要先入 SaturnVoiceData 这个库就是我们现在用的

* 将 md5 和 对应的 position_id 信息入到 TurbineLabelStudio 这个平台，6 个 wav + 点位信息 -> buc_id 

* 将 buc_id 分配给指定数据集（没有数据集就新建）

* 如果有新标签的话 注册新的标签

* 在标注修改页面标注 指定数据集，

* 如果需要训练使用 api 将 img 和 xml 下载到本地


### 后续的展望

* 将六个 wav 合并为一个自定义的文件，方便整理和规范化，代码中处理的时候也方便一些

* 增加一个标签对比的界面，每一个标签听起来是什么样子的需要在不同的 buc之间进行横向对比

* 同一个图片可以随机分配给两个人去标注，然后我们分析，哪些漏标，哪些错标，哪些多标
